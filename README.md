# Сервис прокси-доступа с регистрацией и личным кабинетом

Проект представляет собой комплексное решение для управления доступом к прокси-серверам (виртуальным машинам) на основе одноразовых ключей.

## Технологический стек

* **Backend:** FastAPI, SQLAlchemy (Async), Pydantic
* **База данных:** PostgreSQL 15 (через asyncpg) + Alembic для миграций
* **Асинхронные задачи:** Celery + Redis (брокер сообщений)
* **Отправка email:** aiosmtplib + Mailpit (для локального тестирования)
* **WebSocket:** Встроенные WebSockets FastAPI
* **Frontend:** Vue 3 (Composition API), Vuetify 3, Pinia, Vue Router, Vite
* **Десктопное приложение:** Python + CustomTkinter, asyncio, websockets
* **Инфраструктура:** Docker + Docker Compose
* **Тестирование:** pytest + httpx + pytest-asyncio

## Как запустить проект

Проект полностью контейнеризирован. Для запуска достаточно выполнить одну команду в корневой директории проекта:

```bash
docker-compose up --build -d
```

При запуске FastAPI сервис **автоматически** применит все миграции Alembic (`alembic upgrade head`), так что база данных развернется без дополнительных действий.

**Доступные сервисы после запуска:**
* **Web-интерфейс (Frontend):** http://localhost
* **API & Swagger UI:** http://localhost:8000/docs
* **Почтовый сервер (Mailpit):** http://localhost:8025

## Инструкция по использованию

### 1. Как зарегистрироваться и получить ключ

1. Откройте веб-интерфейс: http://localhost
2. Перейдите на страницу **Регистрация** и создайте аккаунт.
3. После успешной регистрации откройте локальный почтовый сервер **Mailpit**: http://localhost:8025
4. Найдите письмо с темой "Твой ключ активации". Внутри будет ваш **одноразовый ключ**.
5. Авторизуйтесь на сайте под своими учетными данными и перейдите в **Личный кабинет**. Вы увидите свой ключ. Там же его можно обновить (старый ключ будет удален, новый отправлен на почту).

### 2. Как добавить свободную виртуальную машину 

По умолчанию в базе нет свободных серверов. Чтобы протестировать подключение, необходимо создать ВМ от лица администратора.

1. Откройте Swagger: http://localhost:8000/docs
2. Найдите роут `POST /api/admin/create_vm`
3. В блоке параметров добавьте заголовок `X-Admin-Key` со значением `super-secret-admin-key` (по умолчанию из `.env`).
4. В Request Body передайте тестовые данные:
   ```json
   {
     "name": "test-proxy-1",
     "host": "192.168.1.15",
     "port": 1080,
     "protocol": "socks5"
   }
   ```
5. Нажмите Execute. Теперь в пуле есть свободный сервер.

### 3. Как запустить десктопное приложение

Убедитесь, что у вас установлен Python 3.11+. Десктопное приложение запускается локально (не в Docker).

1. Перейдите в директорию десктопного приложения:
   ```bash
   cd desktop_app
   ```
2. Установите зависимости:
   ```bash
   poetry install
   ```
3. Запустите приложение:
   ```bash
   poetry run python main.py
   ```
4. Вставьте ключ, полученный на почту (или из Личного кабинета), и нажмите **Подключиться**.
5. Приложение свяжется с FastAPI, получит данные ВМ, поднимет **WebSocket** соединение и отобразит статус "Подключено" в реальном времени. Если удалить сервер через БД, статус мгновенно изменится.

## Тестирование 

В проекте реализованы интеграционные автотесты API с проверкой авторизации, регистрации, защиты админских эндпоинтов и предотвращения создания дубликатов юзеров.

Запуск тестов выполняется внутри работающего контейнера API:

```bash
docker exec -it proxy-service-api pytest tests/ -v
```
